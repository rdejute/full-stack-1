## 🔹 React Fundamentals

### React vs ReactJS vs React Native

React, ReactJS, and React Native are closely related, but they are used for different things. **React (or ReactJS)** is a JavaScript library used to build user interfaces for websites. You’ll often see “React” and “ReactJS” used interchangeably, and they basically mean the same thing in most cases.

**React Native**, however, is used to build mobile apps for iOS and Android. It uses the same ideas as React (like components), but instead of rendering HTML in the browser, it renders native mobile elements.

So in simple terms:

* React / ReactJS → Web applications
* React Native → Mobile applications

---

### Is React a Library or a Framework?

React is a **library**, not a full framework.

A library focuses on doing one main job—in this case, building user interfaces. A framework, on the other hand, usually provides a full structure for your entire application, including routing, state management, and more.

With React, you get a lot of flexibility. You can choose your own tools for things like routing or state management instead of being forced into one way of doing things. This makes React easier to customize, especially as your project grows.

---

### HTML vs JSX

HTML is the standard language used to structure content on web pages. JSX is similar in appearance, but it is actually used inside JavaScript when working with React.

The main difference is that JSX allows you to mix JavaScript and HTML-like syntax together. For example, you can insert variables or logic directly into your UI using curly braces `{}`.

JSX is not actually understood by browsers directly—it gets converted into regular JavaScript behind the scenes. This makes it more powerful than plain HTML because it allows dynamic and interactive content.

### Why React is a Strong Choice for This Project

React is a strong choice for this project because it makes building interactive user interfaces much easier. It uses reusable components, which helps keep the code organized and easier to manage.

It also updates the UI efficiently when data changes, which is important for user experience. Since this project involves communicating with a backend and displaying dynamic data, React works well because it connects easily with APIs and handles updates smoothly.

## 🔹 Full-Stack Technology Research

### Alternative Tech Stacks

There are many different tech stacks that can be used to build full-stack applications. For example, the **MEAN stack** (MongoDB, Express, Angular, Node.js) is similar to MERN but uses Angular instead of React.

The **LAMP stack** (Linux, Apache, MySQL, PHP) is a more traditional stack often used for server-rendered websites. Another option is using **Django (Python) with React**, where Django handles the backend and React handles the frontend.

Each stack has its pros and cons, depending on the project and the developer’s preferences.

---

### Why MERN is a Strong Option

The MERN stack (MongoDB, Express, React, Node.js) is a strong option because it uses JavaScript for both the frontend and backend. This makes development simpler since you don’t have to switch between different languages.

MongoDB stores data in a format that works well with JavaScript, and Express/Node.js make it easy to build APIs. React handles the frontend and makes the UI dynamic.

Overall, MERN is a great choice because all parts of the application work well together and are built using the same language.
